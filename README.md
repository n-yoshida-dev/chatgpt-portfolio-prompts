# ChatGPT Projects用 個人開発レビュー運用プロンプト集

ChatGPT Projectsを使って、個人開発や技術学習のレビュー運用を安定させるために作成したプロンプト集です。

## この資料について

このリポジトリは、ChatGPTを単なる質問応答の相手として使うのではなく、個人開発の設計・実装・レビュー・学習整理を継続的に支援してもらうためのプロンプトやテンプレートをまとめたものです。

主な目的は、次の3つです。

- チャットを切り替えても過去の文脈を引き継ぎやすくする
- AIにすぐ答えを出させるのではなく、自分の理解や判断を整理しながら進める
- 設計判断・レビュー観点・学習ログをファイルとして残し、継続的に改善できるようにする

AIにすべてを任せるのではなく、**自分で考えるための壁打ち相手・レビュー相手として使う**ことを前提にしています。

## 使い方

基本的な使い方は、以下の記事で紹介しています。

[ChatGPT Projectsで個人開発のレビュー運用を安定させる仕組みを作った](https://qiita.com/n-yoshida-dev/items/064adc2858ddf40c4b4f)

大まかな流れは次の通りです。

1. ChatGPT Projectsを作成する
2. `00_SystemPrompt_ProjectInstructions.md` の内容をProject指示に設定する
3. 必要な補助プロンプトやテンプレートをProjectに追加する
4. 個人開発や技術学習のテーマごとにチャットを分ける
5. チャットが一段落したら、学習ログやADRに内容を残す
6. 成果物が更新されたら、必要に応じてRepomixなどでProject側の参照情報を更新する

## このプロンプト集の考え方

このプロンプト集では、AIにいきなり完成コードや完成設計を出させることよりも、次のような使い方を重視しています。

- 前提条件を整理してもらう
- 自分の案に対してレビューしてもらう
- 抜け漏れやリスクを指摘してもらう
- 設計判断の理由を言語化する
- 次に確認すべき論点を整理する
- 学習ログやADRとして記録する内容を整理する

そのため、AIの役割は「答えを出す人」ではなく、**考えるための補助線を引く人**に近いです。

## 注意事項

`04_フェーズ1専用学習対象定義.md` と `05_ADRインデックス.md` は、私個人の学習状況や開発テーマに合わせて作成した内容です。

そのまま使うというより、次のように自分用に書き換えて使うことをおすすめします。

- 自分が今取り組んでいる技術テーマに合わせる
- 現在のスキルレベルに合わせる
- 作成中のアプリや成果物に合わせる
- レビューしてほしい観点に合わせる
- ADRとして残したい設計判断に合わせる

また、ここにあるプロンプトやテンプレートを使っても、AIの回答が常に正しいとは限りません。

最終的な判断は自分で行い、必要に応じて公式ドキュメントや実際のコードで確認してください。

## 参考

この運用では、以下のような情報をChatGPT Projectsに持たせることを前提にしています。

- Project指示
- 相談テンプレート
- レビュー観点チェックリスト
- 学習ログ
- ADR
- Repomixでまとめたコードベース情報

チャット履歴だけに頼るのではなく、**長期的に残したい情報をファイルとして管理し、必要な場面でAIに参照させる**ことがポイントです。
